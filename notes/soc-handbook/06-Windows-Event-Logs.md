# Module 06 – Windows Event Logs (SOC Fundamentals)

---

## 1. What are Windows Event Logs?

Windows Event Logs are records of activities that occur inside the Windows Operating System.

These logs record:

* User logins
* Failed login attempts
* System activity
* Application activity
* Network access
* Account changes
* Privilege usage

SOC analysts use these logs to detect:

* Unauthorized access
* Brute-force attacks
* Malware execution
* Privilege escalation
* Suspicious behavior

---

## 2. Why Windows Event Logs are Important in SOC?

Windows Security Logs help SOC analysts:

* Investigate login attempts
* Detect compromised accounts
* Monitor admin activity
* Identify attacker entry point
* Detect brute-force attacks
* Track attacker movement

Most enterprise attacks target Windows systems.

---

## 3. Types of Windows Event Logs

---

### a) System Logs

Contain operating system related events.

Examples:

* System startup
* Driver loading
* Hardware issues

---

### b) Security Logs

Contain authentication and authorization events.

Examples:

* Login success
* Login failure
* Account lockout
* Privilege usage
* Account creation

SOC analysts mainly monitor this log.

---

### c) Application Logs

Contain events generated by applications.

Examples:

* Software errors
* Application crashes
* Service activity

---

### d) Setup Logs

Contain installation related activity.

Examples:

* OS installation
* Windows updates

---

## 4. Important Fields in Windows Event Logs

SOC analysts check:

* Timestamp
* Event ID
* Username
* Source IP Address
* Logon Type
* Process Name
* Account Domain
* Failure Reason

---

## 5. What is an Event ID?

Each Windows activity generates a unique Event ID.

Event ID tells:

* What type of activity occurred

SOC analysts use Event IDs to detect attacks.

---

## 6. Important Security Event IDs for SOC L1

---

### 4624 – Successful Logon

Indicates:

User logged in successfully.

SOC Concern:

Check unusual login time or unknown IP.

---

### 4625 – Failed Logon

Indicates:

Login attempt failed.

SOC Concern:

Multiple 4625 events may indicate brute-force attack.

---

### 4634 – Logoff

Indicates:

User logged out.

---

### 4648 – Explicit Credential Logon

Indicates:

User used alternate credentials.

SOC Concern:

May indicate lateral movement.

---

### 4672 – Special Privileges Assigned

Indicates:

Admin privileges assigned.

SOC Concern:

Possible privilege escalation.

---

### 4688 – Process Creation

Indicates:

New process executed.

SOC Concern:

Malicious process execution.

---

### 4720 – User Account Created

SOC Concern:

Unauthorized account creation.

---

### 4726 – User Account Deleted

---

### 4728 – User Added to Admin Group

SOC Concern:

Privilege escalation.

---

### 4732 – User Added to Local Group

---

### 4740 – Account Lockout

Indicates:

Account locked after multiple failed logins.

SOC Concern:

Possible brute-force attack.

---

### 4768 – Kerberos Authentication Ticket Requested

---

### 4769 – Service Ticket Requested

SOC Concern:

Used in lateral movement.

## 7. What is Logon Type?

Logon Type shows:

How user logged into system.

---

### Common Logon Types

| Logon Type | Description              |
| ---------- | ------------------------ |
| 2          | Interactive Login        |
| 3          | Network Login            |
| 4          | Batch Login              |
| 5          | Service Login            |
| 7          | Unlock Workstation       |
| 8          | Network Cleartext        |
| 10         | Remote Interactive (RDP) |
| 11         | Cached Interactive       |

SOC Concern:

Logon Type 3 or 10 from unknown IP may indicate attack.

---

## 8. Failed Logon Analysis

SOC analysts check:

* Target username
* Source IP
* Logon Type
* Failure Reason
* Time of attempt

Multiple failed attempts from same IP may indicate:

Brute-force attack.

---

## 9. Account Lockout Detection

Event ID:

4740

SOC analysts investigate:

* Username targeted
* Lockout time
* Login attempts before lockout

Indicates password attack attempt.

---

## 10. Privilege Escalation Detection

SOC analysts monitor:

* Event ID 4672
* Event ID 4728
* Event ID 4732

Indicates user gaining admin rights.

---

## 11. Process Creation Monitoring

Event ID:

4688

SOC analysts check:

* Process name
* Parent process
* Command line

Detect:

* Malware execution
* Suspicious scripts

---

## 12. Lateral Movement Detection

SOC analysts monitor:

* Event ID 4648
* Event ID 4769

Indicates:

Accessing other systems using credentials.

---

## 13. What is Event Log Forwarding?

Logs from multiple systems are:

Collected and forwarded to central SIEM.

Used for:

Centralized monitoring.

---

## 14. Windows Event Log Investigation Steps

SOC analysts:

1. Identify Event ID
2. Check username
3. Check source IP
4. Check login time
5. Check logon type
6. Check process activity
7. Correlate related events

---

## 15. Common Attack Indicators in Windows Logs

* Multiple failed logins
* Login from unknown IP
* Account lockout
* New admin account creation
* Suspicious process execution
* Privilege assignment

---

## 16. How Windows Logs Help in Incident Response?

Logs help SOC analysts:

* Identify attacker entry
* Detect compromised accounts
* Track attacker movement
* Understand attack timeline
* Investigate alerts

---

## 17. SOC Use Case Example

Alert:

Multiple failed login attempts detected.

Investigation:

1. Check Event ID 4625
2. Check username
3. Check Source IP
4. Check Logon Type
5. Check Event ID 4740
6. Identify suspicious behavior

Possible attack:

Brute-force attempt.

## 18. Windows Log Storage Location

Windows Event Logs are stored in:

```
C:\Windows\System32\winevt\Logs\
```

File format:

```
.evtx
```

SOC Relevance:

Used in forensic investigations and offline log analysis.

---

## 19. Windows Log Channels

Important log channels SOC analysts should know:

* Security
* System
* Application
* Microsoft-Windows-Sysmon/Operational (if Sysmon installed)
* PowerShell/Operational

SOC Relevance:

Many advanced detections come from Sysmon and PowerShell logs.

---

## 20. Event ID vs Task Category vs Level

Each log entry contains:

* Event ID → Type of event
* Level → Information / Warning / Error
* Task Category → Action category

Interviewers may ask:

“What fields do you check in a Windows Security log?”

---

## 21. What is Audit Policy?

Windows logs are generated based on:

Audit Policies.

Example:

* Audit Logon Events
* Audit Account Management
* Audit Object Access
* Audit Process Creation

If auditing is not enabled → logs will not be generated.

SOC Relevance:

Important in real environments.

---

## 22. Correlation Example (Full Attack Flow)

SOC L1 should understand how events link together:

Example:

1. 4625 → Multiple failed logins
2. 4624 → Successful login
3. 4672 → Admin privileges assigned
4. 4688 → Suspicious process executed

This shows:

Brute-force → Account compromise → Privilege escalation → Malware execution

This is how real SOC investigations work.
