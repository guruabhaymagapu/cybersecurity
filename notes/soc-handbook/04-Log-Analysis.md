# Module 04 – Log Analysis 

---

## 1. What is a Log?

A **log** is a recorded event that happens inside a system, application, or network.

Logs store important information such as:

* Login attempts
* File access
* Network connections
* System errors
* Application activities
* Security events

Logs help SOC analysts understand:

* What happened?
* When did it happen?
* Who performed the action?
* From where did it happen?

---

## 2. Why Logs are Important in SOC?

SOC analysts use logs to:

* Detect security incidents
* Investigate suspicious activity
* Identify unauthorized access
* Detect malware infections
* Monitor user behavior
* Track attacker actions
* Support incident response
* Meet compliance requirements

Logs act as **digital evidence** during investigations.

---

## 3. Types of Logs

### a) System Logs

Contain operating system related events.

Examples:

* System boot
* Driver loading
* Hardware failures

---

### b) Security Logs

Contain authentication and authorization related events.

Examples:

* Login attempts
* Account lockouts
* Privilege usage
* User creation

---

### c) Application Logs

Contain events generated by software applications.

Examples:

* Browser activity
* Database access
* Software errors

---

### d) Network Logs

Contain network traffic related information.

Examples:

* Source IP address
* Destination IP address
* Port numbers
* Protocol used

---

## 4. Log Sources in SOC

Logs are collected from different devices such as:

* Endpoints (Windows/Linux)
* Firewalls
* Routers
* IDS/IPS
* Servers
* Antivirus / EDR systems
* Web servers
* Email servers
* Active Directory
* SIEM tools

SOC analysts monitor logs from all these sources.

---

## 5. Common Fields in Logs

Most logs contain the following important fields:

* Timestamp → When event happened
* Source IP → Sender address
* Destination IP → Receiver address
* Username → Account used
* Event ID → Type of event
* Port Number → Communication port
* Protocol → TCP/UDP
* Action → Allowed or Denied
* Status → Success or Failure
* Severity Level

These fields help analysts during investigation.

---

## 6. Log Severity Levels

Logs are categorized based on importance:

* Informational → Normal activity
* Warning → Possible issue
* Error → System problem
* Critical → Serious issue
* Alert → Immediate action required
* Emergency → System failure

SOC analysts prioritize alerts based on severity.

---

## 7. What is Log Monitoring?

Log monitoring means continuously observing logs to detect:

* Suspicious activity
* Failed login attempts
* Unauthorized access
* Malware behavior
* Data exfiltration

Monitoring can be:

* Real-Time Monitoring (Live detection)
* Historical Analysis (Past investigation)

SOC analysts use SIEM tools for monitoring.

---

## 8. Log Analysis Process

SOC analysts follow these steps:

1. Collect logs from devices
2. Normalize log format
3. Parse log data
4. Filter relevant logs
5. Search suspicious events
6. Correlate events
7. Investigate alerts
8. Report findings

---

## 9. What is Log Parsing?

Log parsing means extracting meaningful information from raw log data.

Example:

From a raw log entry, extract:

* Username
* IP address
* Login status
* Timestamp

Parsing makes logs readable and structured.

---

## 10. What is Log Normalization?

Log normalization is the process of converting logs from different sources into a common format.

Example:

Windows:
EventID = 4625

Linux:
Failed password

Both represent:
Authentication Failure

Normalization allows SIEM tools to analyze logs efficiently.

---

## 11. What is Log Filtering?

Filtering helps analysts:

* Remove unnecessary logs
* Focus on important events
* Reduce investigation time

Examples:

* Filter by IP address
* Filter by username
* Filter by Event ID
* Filter by time range

---

## 12. What is Log Correlation?

Log correlation means connecting multiple log events to identify a security incident.

Example:

* Multiple failed logins
* Same source IP
* Within short time
* Followed by successful login

This may indicate a **Brute-Force Attack**.

---

## 13. Indicators of Suspicious Activity in Logs (IOCs)

SOC analysts look for:

* Multiple failed login attempts
* Login at unusual time
* Login from unknown location
* Account lockouts
* Privilege escalation
* New user account creation
* Unusual outbound traffic
* Unknown process execution

These are known as **Indicators of Compromise (IOCs).**

---

## 14. False Positive vs True Positive

### False Positive

Security alert generated for normal activity.

Example:
Employee enters wrong password multiple times.

---

### True Positive

Alert generated for actual malicious activity.

Example:
Attacker performing brute-force login attack.

SOC analysts must validate alerts before escalating.

---

## 15. What is a Log Alert?

An alert is generated when suspicious activity is detected in logs.

Examples:

* Multiple failed login attempts
* Malware detection
* Unauthorized access
* Privilege escalation

Alerts are investigated by SOC analysts.

---

## 16. Log Retention

Log retention means storing logs for a specific period.

Logs are stored for:

* Incident investigation
* Compliance requirements
* Legal purposes
* Forensic analysis

---

## 17. How Logs Help in Incident Investigation

Logs help SOC analysts:

* Identify attacker entry point
* Track attacker movement
* Detect compromised accounts
* Understand attack timeline
* Identify affected systems

Logs help build a complete attack story.

---

## 18. SOC Use Case Example

Alert Received:
Multiple failed login attempts from same IP.

Investigation Steps:

1. Check username targeted
2. Check source IP address
3. Check login time pattern
4. Look for account lockout
5. Check successful login after failures
6. Check privilege usage
7. Identify suspicious activity

---

## 19. Source IP vs Destination IP in Logs

* Source IP → Origin of traffic
* Destination IP → Target system

Used to identify attacker system and victim system.

---

## 20. Why Ports Matter in Logs?

Port numbers indicate the service used during communication.

Common ports:

* 80 → HTTP
* 443 → HTTPS
* 22 → SSH
* 3389 → RDP
* 25 → SMTP

Unusual port activity may indicate malicious behavior.

---

## 21. Importance of Time Synchronization

Logs from different systems must have synchronized time.

This helps in:

* Accurate incident timeline
* Proper event correlation
* Correct attack reconstruction

Time mismatch can break investigations.

---

## 22. Challenges in Log Analysis

SOC analysts face challenges such as:

* High log volume
* False alerts
* Log noise
* Missing logs
* Complex formats
* Encrypted traffic visibility

Efficient log management is critical for SOC operations.
